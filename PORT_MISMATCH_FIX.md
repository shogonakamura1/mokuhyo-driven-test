# ポート不一致エラーの解決方法

## 問題の原因

**Next.jsの開発サーバーはポート3001で起動していますが、ブラウザはポート3000にアクセスしようとしています。**

```
⚠ Port 3000 is in use, trying 3001 instead.
  - Local:        http://localhost:3001
```

しかし、ブラウザでは`http://localhost:3000`にアクセスしているため、404エラーが発生しています。

## 解決方法

### 方法1: ブラウザで正しいポートにアクセス（最も簡単）

ブラウザで以下のURLにアクセスしてください：

```
http://localhost:3001
```

### 方法2: ポート3000を使用しているプロセスを停止

```bash
# ポート3000を使用しているプロセスを確認
lsof -i :3000

# プロセスIDを確認して停止（例：PIDが47000の場合）
kill -9 47000

# Next.jsの開発サーバーを再起動（ポート3000で起動するはず）
cd apps/web
npm run dev
```

### 方法3: 環境変数でポートを指定

```bash
cd apps/web
PORT=3000 npm run dev
```

ただし、ポート3000が既に使用されている場合は、そのプロセスを停止する必要があります。

## 確認方法

### 1. 正しいURLにアクセスしているか確認

ブラウザのアドレスバーを確認してください：
- ❌ `http://localhost:3000` → 404エラー
- ✅ `http://localhost:3001` → 正常に動作するはず

### 2. Next.jsのログを確認

ターミナルに以下のようなメッセージが表示されているはずです：

```
  - Local:        http://localhost:3001
```

このURLにアクセスしてください。

## 推奨される手順

1. **ブラウザで`http://localhost:3001`にアクセス**
2. 正常に動作することを確認
3. ポート3000を使用しているプロセスを停止（必要に応じて）
4. Next.jsを再起動してポート3000で起動する

## ポート3000を使用しているプロセスを確認する方法

```bash
# ポート3000を使用しているプロセスを確認
lsof -i :3000

# または
netstat -an | grep 3000
```

プロセスを停止するには：

```bash
# プロセスID（PID）を確認して停止
kill -9 <PID>
```
